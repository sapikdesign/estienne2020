Les balises de contenu

En HTML, les balises permettent de distinguer les différents éléments d'une page web en les encadrant entre une balise d'ouverture et une balise de fermeture.
Le nom d'un élément HTML est le nom utilisé dans des chevrons, comme par exemple p pour un paragraphe. Le nom de la balise fermante est précédé par un caractère barre oblique.
Pour les éléments vides (image, saut de ligne, séparateur...), il n'y a pas de balise de fin.

On peut leur attribuer un identifiant (id) et/ou une classe (class) pour pouvoir définir leur apparence en CSS (couleur de texte, police, corps, alignement...).

Les balises de contenu sont détectées et interprétées par les navigateurs, même si aucun style supplémentaire n'est précisé.
Simplement avec ces balises, on peut structurer une page html en distinguant du texte courant, des titres, des listes, des liens...

Le texte

L'élément html p représente un paragraphe de texte. Les paragraphes sont généralement représentés comme des blocs et séparés par un espace horizontal.

Habiller du texte

Dans tous les types de texte (paragraphes, titres, listes...), on peut ajouter des balises pour :
Faire un retour à la ligne, avec br
Mettre du texte en gras, avec strong
Mettre du texte en italique, avec i
Mettre un filet de séparation avec hr

Les titres

Les titres permettent de hiérarchiser le texte, mais servent également au référencement : les moteurs de recherche considèrent que les titres contiennent des mots-clés important, ils scannent donc les pages en commençant par les titres.

Il existe 6 niveaux de titre :

Les liens

Les liens sont encadrés par une balise a, à laquelle on attribue au moins un paramètre : l'adresse vers laquelle doit pointer le lien. On peut ensuite définir un titre, qui apparait au survol du lien, et une cible, pour préciser si le lien doit s'ouvrir dans la même fenêtre (par défaut) ou dans une nouvelle.

Les images

Les images doivent être stockées dans le dossier contenant le site, éventuellement rangées dans un dossier spécifique. On fait ainsi un lien vers l'image, en indiquant en source (src) le chemin entre le fichier html et l'image.
Les formats acceptés sont le jpg, le gif (animé ou non), le png (utilisé pour la gestion de la transparence) et le svg (format vectoriel).
Pour afficher un pdf, il faudra faire un lien vers le fichier, qui s'ouvrira grâce à Adobe Reader.
Le champs alt permet d'ajouter des informations sur l'image, utiles pour son référencement et lues par les navigateurs pour les malvoyants. Si l'image est manquante, c'est le contenu du champs qui sera affiché.
On peut définir la taille de l'image directement dans la balise, en ajoutant les paramètres width (largeur) et/ou height (hauteur) et en indiquant une valeur, en pixels par défaut.

La balise span

Elle ne représente rien de particulier, mais permet d'appliquer un style à du texte, soit directement en ajoutant un style à la suite (cf ci-dessous : les couleurs), soit en lui attribuant une classe qui sera reprise en CSS.
Elle doit uniquement être utilisée lorsqu'aucun autre élément sémantique n'est approprié.

Les couleurs

Elles sont appliquées en encadrant les termes par une balise span, qui permet d'ajouter un style à du texte; ici le style color.
Les couleurs sont définies par différentes unités de valeurs :

  • Valeur héxadécimale, composée de 6 caractères précédés d'un dièse
    Blanc = #ffffff ou #fff // Noir = #000000 ou #000
  • Valeur nominale prédéfinie
    Blanc = white // Noir = black
  • Valeur rgb, qui permet d'ajouter un paramètre d'opacité.
    Blanc = rgb(255,255,255) // Noir = rgb(0,0,0)
Rouge (red)
Bleu (deepskyblue)
Rose (#ff99cc)
Orange (#ffcc99)
Noir avec opacité de 50%

NE PAS OUBLIER

  • Nommer les fichiers sans espace, ni majuscules ni accents ou caractères spéciaux.
    Cela vaut pour les fichiers html mais aussi pour les images.
  • Vérifier que les balises qui doivent l'être sont fermées.
  • Le poids des images doit être optimisé au mieux pour être le plus léger possible, tout en conservant une bonne qualité, pour ne pas impacter négativement sur le temps de chargement de la page.
